#메모리의 이해 (Memory Management)

운영체제는 한정된 주기억장치(RAM) 를 여러 프로그램이 효율적으로 사용할 수 있도록 관리합니다.

    역할: 프로그램(프로세스)이 실행되려면 코드와 데이터를 메모리에 올려야 함.

    문제: 메모리가 부족하면 실행 중인 프로그램이 서로 간섭하거나, 실행이 중단될 수 있음.

    해결: OS는 메모리를 할당, 보호, 해제, 스와핑하는 기능을 수행함.

#주요 개념

    물리주소(Physical Address): 실제 RAM의 주소

    논리주소(Logical Address): 프로그램이 보는 가상 주소

    주소 변환(Address Mapping): 논리주소 → 물리주소 변환 (MMU가 담당)


#메모리의 계층

[레지스터]
    ↓
[캐시(L1/L2)]
    ↓
[메인메모리 (RAM)] : 휘발성 CPU가 직접 접근
    ↓
[보조기억장치 (HDD/SSD)]

#메모리 할당방식
가변분할 / 고정분할
필요한만큼 / 모든 부위에 정량으로

#스와핑

    1.메인메모리에 있는 실행상태의 프로세스
    2.보조장치 스왑 영역에 있는 실행상태가 아닌 프로세스
    간에서 프로세스가 이동하는(교환되는) 것

#외부단편화
    프로세스를 메모리에 연속 배치하다보면, 테트리스가 안되서 비어있지만 낭비되는 공간이 생김.

#메모리의 이해 예제

예시:
프로그램 A, B가 동시에 실행됨.

A: 100MB 필요

B: 200MB 필요
→ RAM이 256MB라면 OS는 일부를 디스크에 잠시 저장(Swap Out)하고 필요할 때 다시 불러옴(Swap In).
