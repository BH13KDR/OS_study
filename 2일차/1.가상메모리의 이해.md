#가상 메모리 관리 (Virtual Memory)

실제 메모리보다 큰 주소 공간을 흉내내는 기술.
→ 실행 중인 프로그램이 전체를 메모리에 올리지 않고도 실행 가능하게 함.

핵심 아이디어: 필요한 부분만 메모리에 올리고, 나머지는 보조기억장치(디스크)에 저장.

장점: 더 많은 프로그램을 동시에 실행 가능.

단점: 디스크 접근은 느리므로 과도하면 “스레싱(thrashing)” 발생.

#주요 개념:

페이지(Page): 메모리를 일정한 크기로 나눈 단위 (예: 4KB)

페이지 테이블(Page Table): 각 페이지의 실제 위치(물리주소)를 기록하는 표

페이징(Paging): 필요한 페이지를 RAM으로 불러오는 과정

세그먼트(Segmentation): 프로그램의 논리적 단위

페이드 세그먼:페이징 + 세그먼트 혼합 : 현대 OS의 표준

    페이지 테이블(연결용 매핑 표) → 논리 주소 ↔ 물리 주소 변환

    세그먼트 단위로 프로그램 구조 유지

    세그먼트 내부는 페이지 단위로 관리 → 단편화 최소화

# 가상 메모리 관리 예제

예: 16KB짜리 프로그램, RAM은 8KB뿐일 때
→ 처음 8KB만 불러오고, 나머지는 필요할 때 페이지 단위로 디스크에서 교체.
